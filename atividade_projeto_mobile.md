# ATIVIDADE — INÍCIO DO PROJETO MOBILE

## Objetivo da atividade

Iniciar o desenvolvimento do projeto da disciplina, estruturando as primeiras definições técnicas e documentais do aplicativo mobile.

A atividade está alinhada ao plano de aprendizagem, contemplando desenvolvimento de solução real, documentação progressiva e impacto sociocomunitário.

## Enunciado da atividade

Cada grupo deverá desenvolver a proposta inicial do aplicativo mobile, contemplando os seguintes itens:

1. Detalhamento técnico do projeto
2. Contexto sociocomunitário
3. Escopo do projeto
4. Requisitos funcionais

## Entrega

Documento em PDF contendo todos os itens solicitados.

## Modelo para desenvolvimento

### 1. Detalhamento técnico

Plataforma, linguagem, framework, ferramentas utilizadas, arquitetura e funcionamento geral do sistema.

### 2. Contexto sociocomunitário

Problema identificado, público-alvo, impacto esperado e benefícios para a comunidade.

### 3. Escopo do projeto

Definir o que o sistema irá fazer e o que não fará (limitações do projeto).

### 4. Requisitos funcionais

Listar funcionalidades do sistema no formato:

- RF01 – descrição
- RF02 – descrição

## Exemplo de projeto

**Nome do projeto:** Conecta Bairro

### Detalhamento técnico

Aplicativo mobile multiplataforma utilizando React Native com Expo. Linguagem JavaScript, estrutura baseada em componentes, armazenamento local e possibilidade futura de integração com API.

### Contexto sociocomunitário

O projeto busca melhorar a comunicação entre moradores de bairros, permitindo registro de problemas locais e organização comunitária.

### Escopo do projeto

- O sistema irá permitir cadastro de usuários, registro de ocorrências e visualização de problemas.
- O sistema não irá realizar pagamentos nem integrar com órgãos públicos neste momento.

### Requisitos funcionais

- RF01 – Permitir cadastro de usuários
- RF02 – Permitir login
- RF03 – Permitir registro de ocorrências
- RF04 – Exibir lista de ocorrências
- RF05 – Visualizar detalhes
